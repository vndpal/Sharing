**Step 1: Verify your Ubuntu environment**

Open your Ubuntu terminal (from Start menu or `wsl` in PowerShell):

```bash
python3 --version
pip3 --version
```

If Python is missing or below 3.7, install it:

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv -y
```

Also make sure Docker Desktop on Windows has WSL integration enabled for Ubuntu:
- Open Docker Desktop on Windows
- Settings, Resources, WSL Integration
- Toggle on for Ubuntu
- Apply & Restart

Verify from Ubuntu:

```bash
docker --version
docker ps
```

**Step 2: Set up your project folder**

Important: work inside the Ubuntu filesystem, not on `/mnt/c/`. Files in `/mnt/c/` (your Windows drive) are 10-20x slower from WSL and cause weird permission issues with `ucc-gen`.

```bash
mkdir -p ~/code
cd ~/code
python3 -m venv .venv-ucc
source .venv-ucc/bin/activate
pip install --upgrade pip
pip install splunk-add-on-ucc-framework
ucc-gen --help
```

If you want to access these files from Windows Explorer or VS Code, just use `\\wsl$\Ubuntu\home\<your-username>\code` in Explorer, or open the folder in VS Code with the "WSL" extension (VS Code will prompt you).

**Step 3: Initialize the new add-on**

```bash
cd ~/code
ucc-gen init \
  --addon-name "Splunk_TA_MyNewAddon" \
  --addon-display-name "My New Addon" \
  --addon-input-name my_input \
  --addon-rest-root my_new_addon

cd Splunk_TA_MyNewAddon
ls -la
```

You should see `globalConfig.json`, `package/`, and supporting files.

**Step 4: Sanity-check build**

```bash
ucc-gen build --ta-version 1.0.0
ls output/
```

If `output/Splunk_TA_MyNewAddon/` appears with `default/`, `bin/`, `appserver/`, your toolchain is working.

**Step 5: Bring in your old code for reference**

If your old broken repo is on the Windows side (e.g. `C:\Users\you\old_addon`), copy it into WSL so diffing is fast:

```bash
cp -r /mnt/c/Users/<your-windows-username>/path/to/old_addon ~/code/old_addon_reference
```

Now you have:

```
~/code/
├── .venv-ucc/
├── Splunk_TA_MyNewAddon/      ← new clean scaffold
└── old_addon_reference/       ← your broken old code, for reference only
```

Open both in VS Code (`code .` from `~/code`) for side-by-side diffing.

**Step 6: Port files one section at a time**

Follow the order from my previous message:

1. Port `globalConfig.json` section by section (meta stays from scaffold, copy `pages.configuration.tabs`, then `pages.inputs.services`, then `pages.dashboard`)
2. After each section, run `ucc-gen build --ta-version 1.0.0` to catch issues
3. Copy `package/bin/my_input.py` from old to new
4. Copy `package/lib/requirements.txt`
5. Copy any `package/default/*.conf` you hand-wrote (your predefined disabled inputs)
6. Copy any custom controls from `package/appserver/static/js/build/custom/`

**Step 7: Run Splunk in Docker with bind mount**

From `~/code/Splunk_TA_MyNewAddon`:

```bash
docker run -d --name splunk-dev \
  -p 8000:8000 -p 8089:8089 \
  -e SPLUNK_PASSWORD=Changeme123 \
  -e SPLUNK_START_ARGS=--accept-license \
  -v $(pwd)/output/Splunk_TA_MyNewAddon:/opt/splunk/etc/apps/Splunk_TA_MyNewAddon \
  splunk/splunk:latest
```

Wait about 60 seconds (`docker logs -f splunk-dev` to watch startup), then open `http://localhost:8000` in your Windows browser. Login: `admin` / `Changeme123`. Your add-on should be in the apps list.

**Step 8: Create a Makefile for the dev loop**

Save this as `Makefile` in `~/code/Splunk_TA_MyNewAddon/`:

```makefile
VERSION ?= 1.0.0
CONTAINER = splunk-dev

build:
	ucc-gen build --ta-version $(VERSION)

restart:
	docker exec $(CONTAINER) /opt/splunk/bin/splunk restart

deploy: build restart
	@echo "Deployed. Hard refresh your browser."

logs:
	docker exec -it $(CONTAINER) tail -f /opt/splunk/var/log/splunk/splunk_ta_mynewaddon_my_input.log

shell:
	docker exec -it $(CONTAINER) bash

splunk-start:
	docker start $(CONTAINER)

splunk-stop:
	docker stop $(CONTAINER)
```

If you don't have `make`:

```bash
sudo apt install make -y
```

Now your dev loop is:

```bash
# Edit source files in VS Code...
make deploy
# Hard refresh browser (Ctrl+Shift+R)
```

To watch input logs in real time in a second terminal:

```bash
make logs
```

**Step 9: WSL-specific gotchas to know about**

- **Line endings**: if you ever copy files from Windows and they have CRLF endings, Python in the Splunk container may complain. Fix with `dos2unix <file>` (install: `sudo apt install dos2unix`).
- **File permissions**: files created on Windows side and viewed from WSL sometimes show as `777`. Usually harmless but can confuse Splunk. Stick to creating/editing inside WSL.
- **Localhost works seamlessly**: `localhost:8000` from your Windows browser hits the Docker container running under WSL. No port forwarding needed.
- **VS Code**: install the "WSL" extension on Windows VS Code. Then in Ubuntu terminal type `code .` and it opens VS Code on Windows attached to your WSL files. Best of both worlds.

**Step 7: Deploy to your local Splunk**

You already did the one-time permissions fix on `C:\Program Files\Splunk\etc\apps\`. Now create `Makefile` in your project root (`~/code/Splunk_TA_MyNewAddon/Makefile`):

```makefile
VERSION ?= 1.0.0
ADDON = Splunk_TA_MyNewAddon
SPLUNK_APPS = /mnt/c/Program Files/Splunk/etc/apps
SPLUNK_BIN = /mnt/c/Program Files/Splunk/bin/splunk.exe

deploy:
	ucc-gen build --ta-version $(VERSION)
	rm -rf "$(SPLUNK_APPS)/$(ADDON)"
	cp -r "output/$(ADDON)" "$(SPLUNK_APPS)/"
	"$(SPLUNK_BIN)" restart
	@echo "Done. Refresh http://127.0.0.1:8000"
```

Install make if missing:

```bash
sudo apt install make -y
```

Run it:

```bash
make deploy
```

Wait for Splunk to finish restarting (about 30-60 seconds). Then open `http://127.0.0.1:8000` in your Windows browser. Your add-on should appear in the Apps dropdown.

**Step 8: Verify the add-on loaded correctly**

Click your add-on from the Apps menu. You should see the Configuration and Inputs tabs from your `globalConfig.json`. Click around, confirm the fields render.

If it doesn't appear or shows errors, check the Splunk log:

```bash
tail -100 "/mnt/c/Program Files/Splunk/var/log/splunk/splunkd.log" | grep -i "Splunk_TA_MyNewAddon\|error"
```

Common issues:
- Malformed `globalConfig.json`: fix the JSON, rebuild
- Missing required field in `app.manifest`: compare with the scaffold's original
- File permission issues: re-do the Windows folder permissions step

**Step 9: Create your first input and watch it run**

In the Splunk UI for your add-on:

1. Go to Configuration tab, set up an account or credentials if your input needs them
2. Go to Inputs tab, click "Create New Input"
3. Fill in the form (name, interval, any custom fields)
4. Save

Now watch the modular input log to confirm it ran:

```bash
tail -f "/mnt/c/Program Files/Splunk/var/log/splunk/splunk_ta_mynewaddon_my_input.log"
```

You should see log lines when the input fires (immediately on save, then every `interval` seconds).

**Step 10: Verify data landed in Splunk**

In Splunk Web, go to Search & Reporting. Run:

```
index=* sourcetype=<your_sourcetype> earliest=-15m
```

(Replace `<your_sourcetype>` with whatever your input writes to.) If you see events, the full pipeline works.
